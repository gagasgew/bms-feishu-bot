仿真与极限测试

### `sim` — 电池模型仿真
切断物理 AFE 采样，使用内部一阶 RC 等效电路数学模型接管 BMS。

| 用法 | 说明 |
|------|------|
| `sim on [soc_pct]` | 开启仿真，默认电芯初始化为 50%。例: `sim on 80`（初始化为 80%） |
| `sim off` | 关闭仿真，将数据源交还给真实 AFE 硬件 |
| `sim soc <cell> <soc>` | 瞬间篡改指定电芯 SOC（引发压差）。cell=255 代表全车。例: `sim soc 0 95`(第0节电池 SOC 改为 95%)。 |
| `sim dyn <cell> <Ro> <Rp> <Cp>` | 动态篡改等效内阻与电容（模拟老化/微短路）。Ro/Rp 单位 0.1mΩ，Cp 单位 F。例: `sim dyn 255 5 5 20000`  (全车内阻改为 0.5mΩ，电容 20000F)。|

### `inj` — 物理量偏移劫持
在真实采集数据基础上强加固定偏移，用于测试保护触发。

| 用法 | 说明 |
|------|------|
| `inj <en> <v_off> <i_off> <t_off> [cell_idx] [temp_idx]` | en: 1开/0关, v_off: 电压偏移(mV), i_off: 电流偏移(mA), t_off: 温度偏移(0.1℃) |

例: `inj 1 -1000 50000 0 5 0` — 开启劫持，第5号电芯强行跌落 1000mV，并模拟放电电流 50A

### `bal` — 硬件均衡测试
强控 AFE 底层的均衡放电开关。

| 用法 | 说明 |
|------|------|
| `bal auto` | 恢复 BMS 的自动奇偶均衡逻辑 |
| `bal override <0xMask>` | 以十六进制掩码强行闭合指定电芯的均衡通道。例: `bal override 0x0003`（强制开启 1 号、2 号电芯均衡） |

> ⚠️ `bal override` 可能引起严重发热，谨慎使用！
